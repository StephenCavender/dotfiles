# Developer Experience Commands

Commands for optimizing developer workflows, tooling, and productivity.

## Commands

### `/dx-audit`
Audit and optimize developer experience, tooling, and workflows. Uses **DX Optimizer** agent.

**Use when:**
- Onboarding takes too long
- Build/test times are slow
- Developers report friction
- Setting up new projects
- Improving team productivity
- Automating repetitive tasks

**Outputs:**
- DX audit report with metrics
- Prioritized improvement roadmap
- Automation scripts and configs
- Updated documentation
- IDE/editor configurations
- Custom CLI tools
- Benchmark comparisons

**Success Metrics:**
- Onboarding time (target: < 5 minutes)
- Build/test execution time
- Developer satisfaction score
- Manual steps eliminated
- Setup error rate

## Examples

```bash
# Audit overall developer experience
/dx-audit Analyze our development workflow and identify bottlenecks

# Optimize onboarding
/dx-audit Review onboarding process and reduce setup time

# Speed up build times
/dx-audit Optimize build and test execution times

# Improve tooling
/dx-audit Audit our development tools and suggest improvements
```
