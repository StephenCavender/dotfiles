Create exhaustive, searchable technical reference documentation that serves as the definitive source of truth.

Reference Documentation Scope:
- Complete API references with all endpoints/methods
- Configuration parameter documentation
- Schema and data model references
- CLI command references
- Error code and status code listings
- Environment variable catalogs
- Type definitions and interfaces

Documentation Structure:

Entry Format:
- Name and signature
- Type and default values
- Required/optional designation
- Version introduced/deprecated
- Comprehensive description
- Parameters with constraints
- Return values and types
- Exceptions and error conditions
- Multiple usage examples
- Cross-references to related items

Organization Requirements:
- Hierarchical structure with deep linking
- Alphabetical index
- Category-based grouping
- Version-specific documentation
- Search-friendly formatting
- Table of contents with navigation

Content Elements:
- Minimal working examples
- Common use case examples
- Advanced configuration examples
- Error handling patterns
- Performance-optimized versions
- Parameter reference tables
- Compatibility matrices
- Feature comparison charts

Quality Standards:
- Every public interface documented
- Verified against actual implementation
- Uniform formatting and terminology
- Consistent naming conventions
- Clear versioning and update tracking
- Searchable with keywords and aliases

Special Sections:
- Quick start guide with copy-paste examples
- Troubleshooting common errors
- Migration guides for version upgrades
- Breaking changes documentation
- Deprecation notices with alternatives

Documentation Types:
- API References (REST, GraphQL, gRPC)
- Configuration Guides (all parameters documented)
- Schema Documentation (fields, types, constraints)
- CLI References (commands, flags, options)
- Error Code Catalogs (codes, causes, solutions)
- Type System Documentation (interfaces, enums, types)

Output Formats:
- Primary: Clean, structured Markdown
- Tables for parameter listings
- Code blocks with syntax highlighting
- Cross-reference links throughout
- Machine-readable metadata (JSON schemas, OpenAPI)

Provide:
- Complete reference with exhaustive coverage
- Organized for quick information retrieval
- Multiple examples per documented feature
- Edge cases, limits, and constraints
- Cross-references to related concepts
- Version history and deprecations
- Troubleshooting guidance

Focus on completeness, precision, and searchability. The goal is to answer every question about the system.
